# Master Question Generator — Category-Agnostic

**Task:** Daily PM Interview Question Generator  
**Schedule:** Every day at 7:00 AM  
**Architecture:** Plugin-based (add new categories without updating this file)

---

## Core Process

### Step 1: Load Configuration
Read `config.yaml` to determine:
- Which categories are active (`active: true`)
- How many questions per category (`questions_per_day`)
- Global settings (difficulty distribution, daily focus rotation, variety rules)

### Step 2: Determine Context
- **Today's date:** [YYYY-MM-DD]
- **Focus index:** `(day_of_month - 1) % 4` → integer in {0, 1, 2, 3}
- **Daily focus:** [monetization, growth, competitive, or mixed] based on focus index

### Step 3: For Each Active Category
For each category where `active: true`:

1. **Load plugin files:**
   - Read `{folder}/generation-rules.md`
   - Read `{folder}/examples.md`

2. **Generate questions:**
   - Generate `{questions_per_day}` questions
   - Follow rules from `generation-rules.md`
   - Apply global variety rules
   - Apply difficulty distribution (60% medium, 30% hard, 10% very hard)
   - Apply daily focus if applicable

3. **Save to file:**
   - Path: `questions/{category_id}/DAY-MM-DD.md`
   - Include metadata (frontmatter)
   - Include framework path
   - Include practice instructions

---

## Question File Template

Every generated question file must follow this structure:

```markdown
---
date: {YYYY-MM-DD}
category: {category_id}
category_name: "{category_name}"
framework: {framework_name}
framework_path: /plugins/{category_id}/{framework_file}
time_limit: {time_limit}
difficulty: {medium | hard | very_hard}
focus: {monetization | growth | competitive | mixed}
---

# {Category Name} Questions - {Day}, {Month DD, YYYY}

**Category:** {Category Name}  
**Generated:** {Timestamp}

---

## Question 1

{Full question following format from generation-rules.md}

**Framework Location:** `/plugins/{category_id}/{framework_file}`

---

## Question 2

{Full question following format from generation-rules.md}

**Framework Location:** `/plugins/{category_id}/{framework_file}`

---

## Practice Instructions

**Before You Start:**
1. Review the recommended framework at the location above
2. Set a timer for {time_limit} minutes
3. Have paper/whiteboard ready for diagrams

**During Practice:**
- Track your time (framework specifies time allocation per section)
- Draw matrices/diagrams as needed
- Speak your answer out loud or type it

**After Practice:**
- Upload this file to the PM Interview Practice Arena (Cowork project)
- The evaluator will assess your answer against the framework rubric
- Review feedback and save your evaluated answer to:
  `/questions/{category_id}/ANSWER-MM-DD.md`

---

*Generated by Cowork Master Generator v1.0*  
*For evaluation, use the PM Interview Practice Arena project*
```

## Global Variety Rules

Apply these across all categories:

### Rule 1: No Repeats
- Don't use the same company/product 2 days in a row
- Track last 2 days of questions across all categories
- If "Instagram" was used yesterday, don't use it today

### Rule 2: Product Mix
For categories that use products (roadmap, portfolio, tech, product-sense):
- 40% Consumer apps
- 30% B2B products
- 30% Indian companies

### Rule 3: Difficulty Distribution
- 60% Medium difficulty
- 30% Hard difficulty
- 10% Very Hard difficulty

### Rule 4: Daily Focus
Based on a 4-day cycle, indexed by `(day_of_month - 1) % 4`:
- **Index 0** (days 1, 5, 9, 13, 17, 21, 25, 29): 70% monetization-focused questions
- **Index 1** (days 2, 6, 10, 14, 18, 22, 26, 30): 70% growth-focused questions
- **Index 2** (days 3, 7, 11, 15, 19, 23, 27, 31): 70% competitive-focused questions
- **Index 3** (days 4, 8, 12, 16, 20, 24, 28): Mixed (balanced)

Note: This applies to roadmap, portfolio, and tech categories. Conceptual and product-sense have their own focus logic.

## Execution Workflow

### 1. Initialization
```
Today: May 22, 2026
Focus index: (22 - 1) % 4 = 1
Focus: Growth (70% growth-focused)

Load config.yaml
Active categories: 
  - product-roadmap (2 questions/day)
  - portfolio (2 questions/day)
  - tech-application (2 questions/day)
  - conceptual (2 questions/day)
  - product-sense (2 questions/day)
  - goals-metrics (1 question/day)
  - debugging (1 question/day)
  - tradeoffs (1 question/day)

Total: 13 questions to generate
```

### 2. For Each Category

#### Example: Product Roadmap

```
Category: product-roadmap
Questions per day: 2
Time limit: 25 minutes

Load:
  - /plugins/product-roadmap/generation-rules.md
  - /plugins/product-roadmap/examples.md

Generate Question 1:
  - Pick product from pool (not used yesterday)
  - Apply daily focus (e.g., focus index 1 → 70% growth-focused)
  - Pick difficulty (roll: 60% chance medium, 30% hard, 10% very hard)
  - Follow template from generation-rules.md
  - Include all required context
  
Generate Question 2:
  - Pick different product (variety rule)
  - Follow same process
  
Save to:
  /questions/product-roadmap/DAY-05-22.md
```

Repeat for all active categories.

### 3. Post-Generation

Verify each file:
- [ ] Frontmatter present with all fields
- [ ] 2 questions included (or per category's questions_per_day)
- [ ] Framework path is correct
- [ ] Practice instructions included
- [ ] No duplicate companies/products from yesterday

## Error Handling

### If plugin file is missing:
```
Error: Could not load /plugins/{category}/generation-rules.md
Action: Skip this category, log error, continue with others
```

### If config.yaml is missing:
```
Error: config.yaml not found
Action: Cannot generate questions, halt with error message
```

### If no categories are active:
```
Warning: No active categories in config.yaml
Action: No questions generated, log warning
```

## Adding New Categories

To add a new category:

1. **Create plugin folder:**
   ```
   /plugins/{new-category}/
     ├── framework.md
     ├── generation-rules.md
     └── examples.md
   ```

2. **Add to config.yaml:**
   ```yaml
   - id: {new-category}
     name: "{Display Name}"
     active: true
     questions_per_day: 2
     time_limit: {X}
     folder: "plugins/{new-category}"
   ```

3. **Create question folder:**
   ```
   /questions/{new-category}/
   ```

4. **Done!** Master generator will automatically:
   - Detect the new category
   - Load its generation rules
   - Generate questions
   - Save to the correct folder

**No changes to this file needed.**

---

## Disabling Categories

To temporarily stop generating questions for a category:

1. Open `config.yaml`
2. Change `active: true` to `active: false`
3. Save

Next run will skip that category.

---

## Testing New Plugins

Before activating a new category:

1. Set `active: false` in config
2. Manually test by calling this generator with only that category active
3. Verify:
   - Questions follow template
   - All context included
   - Framework path correct
   - Difficulty distribution applied
4. Once verified, set `active: true`

## Maintenance

This file should **NEVER** need updating for:
- Adding new categories (use config.yaml)
- Changing question formats (edit plugin's generation-rules.md)
- Updating frameworks (edit plugin's framework.md)
- Adjusting difficulty (edit plugin's generation-rules.md)

This file **ONLY** needs updating for:
- Changes to core generation logic
- Changes to frontmatter structure
- Changes to global variety rules
- Bug fixes in the core process
